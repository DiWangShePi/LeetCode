### Problem

给定两个大小分别为`m`和`n`的正序数组`nums1`和`nums2`，找出并返回这两个正序数组的中位数。

算法的时间复杂度应该为`O(log(m+n))`

###### Example-I

```
输入：nums1 = [1, 3], nums2 = [2]
输出：2.0
```

###### Example-II

```
输入：nums1 = [1, 2], nums2 = [3, 4]
输出：2.50
```

#### Solution I

一个非常自然的（符合时间复杂度要求的前提下）想法，是采用二分查找的形式。针对两个有序数组，分别维护一个指向两个数组中位数的指针，移动这两个指针，保持两个指针（包括指针所指向的元素）所囊括的左半边区域的元素个数等于右半边元素个数（在`n+m`为奇数时多1个），最终找到一个特定的位置，使得`nums1`右边区域最小的元素大于`nums2`左边区域最大的元素，且`nums1`左边区域最大的元素小于`nums2`右边区域最小的元素。这一条件可以被等价视为，在`nums1`中找到最大的元素下标，使得在`nums1`中的前一个元素值小于等于相应的`nums2`元素中的值。

由于需要维护区域的大小，即当`nums1`指针所指向的元素下标确定时，`nums2`指针所指向的元素下标也被相应的确定了。因此二分查找只需要审查`nums1`的部分。
